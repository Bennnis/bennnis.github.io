Slim

Centralized State Management With Proxy, State-Non-Editable.

QuickStart →

State-Readonly

Don't worry about the state's unpredictable mutation because the state can only be modified inside the reducer.

Single-Source

The state is centrally stored and is easy to initial in the isomorphic application.

Easy-Integrated

State hooks are easy to integrate quickly into mainstream frameworks such as Vue.

# Usage

# Installation

# Cdn

<script src="https://unpkg.com/slim-store@1.0.0/slim.min.js"></script>

# Npm

npm install slim-store

# Use in code

// state is single object
import {createStore} from 'slim'

const state = {
    count: 0
}

// reducers is event proxy
const reducers = {
    increment: (state) => {
        state.count += 1
    }
}

// create store
const store = createStore({
    reducers,
    state
})

// emit increment reducer
store.dispatch('increment')

console.log(store.getState().count)
// output: 1
